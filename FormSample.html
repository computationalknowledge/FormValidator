form operation: browser sends data to server
either with post or get encoding
server Operating System with its TCPIP stack gives your form data
to STDIN standard input PIPE to the program specified in the action attribute of the form
the program runs and delivers its output as HTML to STDOUT handle of the server computer's operating system, and that HTML is received
by the request client computer and loaded into browser to displayed

get encoding means that the URL which the browser
sends to the server has the field names and field values
written into the URL as a series of name value pairs POST sends the data into TCP : It is not visible. You can use

Fiddler to do packet sniffing to visualize the TCP packets
and their data payload contents
